Нужно исправить баг повторного прохождения квиза и изменить порог прохождения урока.

ПРОБЛЕМА 1 — квиз обрывается после первого ответа

Сейчас происходит ошибка:

1. Пользователь проходит квиз до конца.
   Например результат: 2 из 5.

2. В попапе урока правильно показывается прогресс (40%).

3. Пользователь запускает квиз снова.

4. После первого ответа (правильного или неправильного)
   квиз сразу заканчивается и показывает экран:

   "Попробуйте ещё раз. Вы выполнили 1 из 5."

Это ошибка.

Правильное поведение:

При повторном запуске пользователь должен иметь возможность
пройти ВСЕ вопросы квиза заново (например 5 из 5 или 12 из 12).


ПРИЧИНА

Скорее всего при старте нового прохождения
не сбрасываются переменные run-состояния.

Например:

currentQuestionIndex
runAnswered
runCorrect


НУЖНО СДЕЛАТЬ

Когда пользователь запускает квиз (navigate("/lesson-quiz")):

полностью сбрасывать run-state:

currentQuestionIndex = 0
runAnswered = 0
runCorrect = 0

Квиз должен позволять пройти все вопросы заново.


BEST PROGRESS

Прогресс урока (например 2 из 5) должен сохраняться,
но НЕ должен ограничивать прохождение текущего квиза.


XP ЛОГИКА

Если вопрос уже был решён правильно раньше:

XP повторно не начисляется.

Если пользователь отвечает правильно на новый вопрос:

XP начисляется.



ПРОБЛЕМА 2 — изменить порог прохождения урока

Сейчас минимальный порог прохождения урока = 60%.

Нужно изменить его на:

PASS_THRESHOLD = 0.5

То есть 50% правильных ответов.


ПРИМЕРЫ

5 вопросов → нужно минимум 3  
12 вопросов → нужно минимум 6  
14 вопросов → нужно минимум 7


ТЕКСТ НА ЭКРАНЕ FAIL

Сейчас текст фиксированный.

Нужно сделать динамический текст.

Пример:

"Вы выполнили 2 из 5 заданий.
Чтобы пройти урок нужно хотя бы 3 из 5."


То есть система должна считать:

requiredCorrect = ceil(totalQuestions * 0.5)

и подставлять это число в текст.



ВАЖНО

Не изменять:

UI
CSS
layout
визуальные компоненты
дизайн страниц


Можно изменять только:

LessonQuizPage.tsx
LessonContext.tsx
UserContext.tsx (если используется для логики прогресса)


ЦЕЛЬ

1. Повторный запуск квиза должен позволять пройти все вопросы.
2. Порог прохождения урока = 50%.
3. Текст результата должен показывать динамическое значение:
   например "3 из 5", "6 из 12", "7 из 14".
