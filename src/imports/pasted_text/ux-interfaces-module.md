Модуль 2 — UX интерфейсов

Урок 1 — Что такое UX
🎯 UX — опыт пользователя
UX (User Experience) — это опыт человека при использовании продукта.
UX показывает:
насколько легко пользоваться интерфейсом


насколько быстро можно выполнить задачу


насколько понятны действия.


UX — это не только дизайн экрана.
 Это структура интерфейса, логика действий и поведение элементов.
Хороший UX делает продукт понятным и удобным.

👤 Пользовательский опыт
Когда пользователь открывает интерфейс, он действует очень быстро:
сканирует экран


ищет знакомые элементы


пытается понять, что можно сделать.


Если интерфейс соответствует ожиданиям, пользователь начинает пользоваться им почти сразу.
Если нет — возникает путаница.

🎯 UX и задачи пользователя
Люди используют продукт ради результата.
Например:
купить товар


заказать услугу


пройти курс


найти информацию.


UX-дизайн должен помогать достичь цели максимально быстро.
Хороший интерфейс:
показывает главное действие


убирает лишние шаги


направляет пользователя к результату.



✨ Простота интерфейса
Каждый элемент интерфейса требует внимания.
Чем больше элементов:
тем сложнее ориентироваться


тем больше времени нужно на решение.


Это называется когнитивная нагрузка.
Хороший UX снижает её:
показывает только нужные элементы


выделяет главное действие


убирает лишнее.



⚠️ Проблемы UX
Плохой UX возникает, когда интерфейс мешает пользователю выполнить задачу.
Например:
слишком много шагов


непонятные кнопки


перегруженный экран


нелогичная структура.


Такие проблемы создают фрустрацию.
В результате пользователь может уйти из продукта.

Урок 2 — User Flow
🧭 User Flow — путь пользователя
User Flow — это последовательность действий пользователя в продукте.
Это путь от входа в интерфейс до результата.
Например:
1 открыть товар
 2 добавить в корзину
 3 оформить заказ.
Хороший User Flow:
короткий


понятный


предсказуемый.



🎯 Цель пользователя
Каждый сценарий должен строиться вокруг цели пользователя.
Пользователь должен быстро понять:
что здесь можно сделать


как это сделать


что произойдёт дальше.


Когда цель ясна, пользователь быстрее принимает решения.

🪜 Шаги пользователя
Любая задача в интерфейсе состоит из шагов.
Каждый шаг требует:
внимания


времени


принятия решения.


Поэтому UX-дизайн старается уменьшить количество шагов.
Чем короче путь — тем выше вероятность, что пользователь завершит действие.

⚠️ Ошибки сценариев
Проблемы возникают, когда сценарий:
слишком длинный


содержит лишние действия


имеет запутанные переходы.


Такие сценарии создают трение (friction).
Задача UX — сделать путь максимально простым.

Урок 3 — Структура интерфейса
🧱 Экран как система блоков
Любой экран можно представить как набор информационных блоков.
Например:
заголовок


описание


карточки


кнопки


формы.


Блоки помогают организовать информацию.
Когда структура понятна, пользователь быстрее читает интерфейс.

📦 Группировка информации
UX использует принцип близости элементов.
Если элементы связаны между собой, они должны:
находиться рядом


иметь одинаковые отступы


образовывать единый блок.


Так пользователь быстрее понимает смысл интерфейса.

🧭 Логика экрана
Информация должна располагаться в естественном порядке чтения.
Например:
1 заголовок
 2 описание
 3 цена
 4 кнопка действия.
Такой порядок помогает пользователю быстро понимать интерфейс.

👀 Сканирование экрана
Люди редко читают интерфейс полностью.
Обычно они сканируют экран глазами.
Исследования показывают два распространённых паттерна чтения:
📖 F-pattern
F‑pattern — пользователь читает экран сверху вниз и слева направо.
Это часто происходит на:
новостных сайтах


страницах с текстом


длинных интерфейсах.



⚡ Z-pattern
Z‑pattern — пользователь движется глазами по форме буквы Z.
Так читаются:
лендинги


простые страницы


промо-экраны.


Поэтому важные элементы размещают по траектории взгляда пользователя.

Урок 4 — Основные элементы интерфейса
🔘 Действия
Кнопки показывают какое действие можно выполнить.
В интерфейсе обычно есть:
главное действие


второстепенные действия.


Главное действие должно:
выделяться визуально


быть самым заметным


иметь понятное название.



⌨️ Ввод данных
Формы позволяют пользователю вводить информацию.
Чтобы форма была удобной:
у каждого поля должна быть подпись


количество полей должно быть минимальным


структура формы должна быть простой.


Чем проще форма, тем выше вероятность заполнения.

🧭 Навигация
Навигация показывает структуру продукта.
Она помогает ответить на три вопроса:
где я нахожусь


куда могу перейти


как вернуться назад.


Хорошая навигация делает интерфейс предсказуемым.

📦 Информационные блоки
Карточки и контейнеры помогают структурировать информацию.
Например:
карточки товаров


блоки курса


секции страницы.


Когда информация разделена на блоки, пользователь быстрее понимает экран.

Урок 5 — Ошибки UX
⚠️ Сложный сценарий
Слишком длинные сценарии — частая ошибка UX.
Когда пользователь должен пройти много шагов:
возрастает вероятность ошибки


пользователь может устать


процесс может прерваться.


Поэтому хороший UX минимизирует количество действий.

❓ Непонятная логика
Интерфейс становится сложным, когда пользователь не понимает результат действия.
Например:
непонятные кнопки


скрытые функции


неожиданные переходы.


Пользователь должен предсказывать результат действия.

🧩 Плохая структура
Когда элементы расположены хаотично:
нет группировки


нарушена иерархия


элементы перемешаны.


Такой интерфейс сложно читать.
Хорошая структура помогает пользователю быстро ориентироваться.

📉 Перегруженный экран
Слишком большое количество элементов создаёт визуальный шум.
Человек не понимает:
куда смотреть


что важнее


какое действие главное.


Лучший интерфейс — простой и сфокусированный.
