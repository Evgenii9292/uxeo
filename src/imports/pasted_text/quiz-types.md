Нужно добавить два новых типа квизов в существующую систему квизов.

Важно:  
не менять существующие квизы и их стили.  
Нужно создать два новых формата, используя текущую архитектуру и UI-компоненты.


---

# 1. Тип квиза: Соединение элементов (Matching)

Задача пользователя:  
соединить элементы интерфейса с их названием.

Пример:  
Toggle → Переключатель  
Checkbox → Чекбокс  
Tooltip → Подсказка


## Поведение

Пользователь нажимает на точку под элементом →  
тянет линию →  
соединяет с правильным вариантом.

Линия должна:

• появляться при drag  
• быть пунктирной  
• фиксироваться при соединении


Если соединение неверное:

линия становится красной  
и появляется возможность переподключить.


Если соединение верное:

линия становится зеленой  
и фиксируется.


Когда все элементы соединены правильно →  
активируется кнопка Продолжить.


## UI

Использовать layout из переданного фрейма.

Слева — элементы интерфейса  
справа — названия.

Точки соединения — маленькие круги.


## Источники компонентов

• карточки — из существующих квизов  
• отступы — из текущего layout  
• checkbox/toggle/tooltip элементы — из существующей UI библиотеки  
• линии соединения — новые


---

# 2. Тип квиза: Multiple Choice (несколько правильных)

Пользователь может выбрать несколько вариантов.

Каждый вариант — карточка с:

• текстом  
• чекбоксом справа.


## Поведение

Клик по карточке:

• выделяет карточку  
• активирует чекбокс  
• цвет чекбокса становится оранжевым.


Можно выбрать несколько.


После нажатия Проверить:

верные ответы → зеленая рамка  
неверные → красная рамка.


После проверки появляется кнопка Продолжить.


## UI

Использовать существующий стиль карточек квиза.

Чекбокс взять из текущей библиотеки UI.

Отступы и размеры — как в существующих квизах.


---

# 3. Общие правила для новых квизов

Использовать существующую систему:

• XP  
• streak  
• progress bar  
• кнопку "Продолжить"


Новые квизы должны работать с той же системой оценки, что и текущие.


---

# 4. Архитектура

Добавить два новых типа:

quiz_type = "matching"  
quiz_type = "multiple_select"


Система квизов должна поддерживать:

single_choice  
matching  
multiple_select


---

# 5. Важно

Не менять текущие стили.

Макет, отступы и layout брать из переданного фрейма.

Использовать существующие UI компоненты.
